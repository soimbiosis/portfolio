PictureThis

Solo Term Project for 15-112: Fundamentals of Programming

In Brief

PictureThis takes a text file and builds a photo collage that represents it. This is achieved by querying the Flickr photo database with each word (that is not in a stopwords list), and placing it in either the background, mid ground, or foreground of a montage based on the frequency with which it appears in the text file. Images in the foreground are larger than images in the mid-ground or background.

Collages generated for A Visit from St. Nicholas, the Wikipedia article about churches, and a New York Times article entitled Iraqi Government and Kurds reach Deal to Share Oil Revenues

Process

Step 1: Word Frequency Analysis In State 1, the user is asked to input a path to a text file. This file is then read, iterated through, and each word is added to a dictionary where the key is the word, and the value is its frequency. Every subsequent occurrence of the word adds to the frequency. Words cannot be in the stopwords list, which consists of very common, non-salient words like ‘the’ and ‘which,’ and they also cannot contain special characters. Finally the dictionary is sorted in reverse frequency order and read into a list so that it can be iterated through in order in the next step.

Step 2: Flickr Image Retrieval In State 2, the bulk of the work is done. I loop through the list of words, and for each word query the Flickr API which returns an xml result from which a url can be constructed. The urls are read into a list, which is then iterated through. Each url is downloaded to a file which is stored in a temporary directory, and is assigned to its respective batch and given a position based on its frequency.

Step 3: Creating the Collage The collage consists of three ‘batches,’ which are groups of pyglet sprites. In State 3, the three batches are drawn to the screen, creating a montage. The user has the option to save the collage by pressing ’s’, or create a new collage by pressing ‘r’.
