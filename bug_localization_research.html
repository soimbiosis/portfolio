Research in Natural Language Processing

Project Description

This was a research project in Natural Language Processing. We worked on developing and evaluating techniques to locate problematic code from natural language bug reports. From the abstract of Automating Extraction of Code Island Queries for Bug Localization, the paper summarizing our process and results:

As software systems continue to grow and evolve, locating code for software maintenance tasks becomes increasingly difficult. Numerous automated bug localization approaches have been proposed. These IR-based techniques typically rely on using words from the titles or descriptions as queries. In this paper, we propose an approach to automatically extracting salient query terms from the title and description by identifying code islands and nouns within the natural language descriptions. Our initial results show that our approach is comparable to human-crafted queries, and can outperform using titles or descriptions alone. More research is needed to generalize these results to other bug repositories.

Project Members

I was mentored by Dr. Emily Hill of Montclair State University. Parallel contributors to this project included Sowmya Bandela and Cynthia Alvarez.

Contribution

My contributions inlcuded proposing and experimenting with different query selection approaches. This involved using srcML to parse titles, method names, and code islands from Java files, the Stanford Part-of-Speech tagger to tag them, Bash and Perl scripting to write them out to files and generate queries, and Apache Lucene to perform query retrieval from source code.
